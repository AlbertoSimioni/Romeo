3.1. Creation of a dataset

To create a new dataset inside the program user has to click the icon dedicated to a dataset's creation from the toolbar. After that user has to insert all required dataset's fields.

3.2. Opening of a dataset

To open an existing dataset double-click one of them from the left panel called Datasets, this action will display the chosen dataset in the central area dedicated to its management.

3.3. Management of a dataset

The program offers a large number of functionalities related to a dataset's management. They concern subjects management, dataset's protocols management, protocols executions, protocols results and datasets elimination.

3.3.1. Insertion of a subject

New subjects can be inserted into a dataset clicking the button New situated at the top of the panel Subjects and defining required subject’s fields.

3.3.2. Elimination of a subject

Removal of subjects from a dataset can be done clicking the button Delete situated at the top of the panel Subjects.

3.3.3. Selection of analyzable subjects

User can select which subjects from a dataset will be analyzed by a protocol's execution clicking their associated checkboxes.

3.3.4. Association of a new protocol

New protocols can be associated to a dataset clicking the button Associate situated at the top of the panel Protocols and selecting them from the displayed list.

3.3.5. Elimination of a protocol

Removal of protocols from a dataset can be done clicking the button Delete situated at the top of the panel Protocols.

3.3.6. Execution of a protocol

Execution of a dataset's protocol is achievable by Execute panel. User has to define some fields related to the analysis (like the path of results folder or export format). He has also three optional fields which permits him to watch results at run-time, to store calcuted features and to watch even features at run-time. After inserting required fields, user can start the execution clicking the button Start Analysis. A new window called Analyzing appears showing percentage of execution and results (only if visualization results option was chosen). Clicking the button Cancel user can abort execution. Clicking the button No Feature user can stop features visualization.

3.3.7. Visualization of a result

To visualize results folder, user has to expand dataset's protocols and, after that, he has to double-click one of their associated results.

3.3.8. Elimination of a dataset

User can eliminate a dataset from the program clicking the icon dedicated to this function from the toolbar.

3.4. Management of protocols list

User can visualize protocols properties and add new ones defined by him inside this list. He can also modify or remove test protocols and turn them into official ones.

3.4.1. Insertion of a new protocol

To insert a new protocol click the icon dedicated to this function from the toolbar and define required protocol’s fields.

3.4.2. Visualization of protocol properties

To visualize properties of a protocol double-click the desired one from the panel Protocols List.

3.4.3. Modification of a test protocol

A test protocol can be modified double-clicking a protocol from the panel Protocols List and redefining its properties.

3.4.4. Elimination of a test protocol

A test protocol can be removed from the panel Protocols List clicking the icon from the toolbar.

3.4.5. Transformation of a test protocol in an official one

A test protocol can be turned into an official one double-clicking a protocol from the panel Protocols List and clicking the checkbox Test.

3.5. Management of features list

This list contains all available feature extractors that can be used for analysis. User, after opening the list, can select a feature extractor to have its description displayed and can also insert into it a new one defined by him.

3.5.1. Opening of features list

To open feature extractors list click the icon from the toolbar.

3.5.2. Insertion of a new feature

To insert a new feature extractor inside this list click the icon from the toolbar and define required fields, like the function name or the path indicating which file system’s file contains compiled code of the new feature.

3.6. Management of algorithms list

This list contains all available clustering algorithms that can be used for analysis. User, after opening the list, can select an algorithm to have its description displayed and can also insert into it a new one defined by him.

3.6.1. Opening of algorithms list

To open clustering algorithms list click the icon from the toolbar.

3.6.2. Insertion of a new algorithm

To insert a new clustering algorithm inside this list click the icon from the toolbar and define required fields, like the function name or the path indicating which file system’s file contains compiled code of the new algorithm.
