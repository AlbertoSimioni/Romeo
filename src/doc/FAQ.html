6.1. What is a "dataset"?

A dataset is a collection of subjects, that are entities represented by an image or a video and an optional mask, that is an image which delimits the region of the other image that has to be analyzed. The dataset represents an aggregate of homogeneus subjects, that is a collection on which you are interested in doing some analysis.

6.2. How can I add a new subject to my dataset?

Once you have created your dataset and it is appeared in the datasets list, double-click on it to select it. In the central part of the screen the contents of your dataset will be listed. Click the New button below the written Subjects in the centre of the screen: a window will ask you to select an image or a video from the filesystem for selecting the subject to analyze, and you will also have the possibility to choose a file representing your mask.

6.3. What is a "protocol"?

A protocol can be seen as a series of analysis that has to be computed on an arbitrary number of subjects. In particular, a protocol can be divided in two parts:

  • - The first part in which you can specify which features you want to extract from the subjects: you can select how many features as you want
  • - The last part in which you can specify a clustering algorithm: the algorithm can be executed on the image directly, if there are no features extracted, or on the set of features extracted. You can specify only one clustering algorithm.

A protocol must have at least one feature to extract or at least one clustering algorithm to apply.

6.4. How can I create a new protocol?

In the main window of the program, looking at the top of the window, you can find the button New Protocol: clicking it you will be sent to a wizard that will allow you to choose the features to extract and the clustering algorithm to apply. In the wizard you will also have the possibility to handle the values of the clustering algorithm parameters if you want to change them.

6.5. How can I execute a protocol?

To execute a protocol follow next steps:

  • - You have first to associate it to the dataset on which you want to execute it.
  • - After having selected the dataset, in the right part of the window you will find the Protocols panel: click on the button Associate and you will see the list of the current defined protocols.
  • - Select one of them and press Ok to associate it to the current dataset.
  • - Once you managed to associate the protocol to the current dataset, just select it in the top right panel Protocols and go below in the Execute panel: here you must specify the Result Path to say to the program where to save the outputs of the analysis. You can also select the output format of the produced images, the vision of the analysis results directly, the vision of the feature extractions and the possibility to save also the features extracted.
  • - Just click on the Start Analysis button to start executing the protocol.

6.6. What do you mean with "Adding a feature extractor or a clustering algorithm"?

ROMEO is structured in order to execute user defined algorithms or feature extractors: the idea is to ensure the possibility to test the extraction of a new feature or the execution of a new algorithm without problems, granting the extensibility of the program.

6.7. How can I define a new feature extractor or a new clustering algorithm?

In the developer user manual of ROMEO, you can find a detailed description of the procedure for the creation of a new feature extractor or of a new clustering algorithm.

6.8. How can I add my feature extractor to ROMEO?

In the top part of the ROMEO window you can find the button Add Feature: click it and you will find a new window in which you have to specify:

  • Feature name: write the name of your new feature
  • Function name: write the name of the function defined in the .h and .cpp files. Be careful to insert only the name of the function (no parameters and to return type)
  • Type: select the category at which your feature belongs
  • Path: type the path to the .dll or .so file created as shared library from your .h and .cpp files
  • Description: type a description of the feature (optional)

After pressing Ok, you will be able to add your feature to a protocol.

6.9. How can I add my clustering algorithm to ROMEO?

In the top part of the ROMEO window you can find the button Add Algorithm: click it and you will find a new window in which you have to specify:

  • Algorithm name: write the name of your new algorithm
  • Function name: write the name of the function defined in the .h and .cpp files. Be careful to insert only the name of the function (no parameters and to return type)
  • Path: type the path to the .dll or .so file created as shared library from your .h and .cpp files
  • Description: type a description of the algorithm (optional)
  • Parameters: here you can define additional parameters that have to be passed to the function representing your algorithm. Normally, the only parameter defined is the number of cluster, but if you need more parameters for your algorithm just press the green plus button to add the desired number of parameters. Make sure to choose the correct type of these parameters.

After pressing Ok, you will be able to add your algorithm to a protocol.

6.10. How can I delete a user defined feature extractor or a clustering algorithm from ROMEO?

You cannot delete feature extractors/clustering algorithms inserted in ROMEO, so be careful in what you decide to insert.

6.11. Why can't I find my image/video from the filesystem browser?

Make sure your image/video corresponds to one of these formats:

  • - JPEG (.jpg)
  • - PNG (.png)
  • - BMP (.bmp)
  • - TIF (.tif or .tiff)
  • - DICOM (.img or .hdr)
  • - NIFTI (.nii)
  • - AVI (.avi)
  • - MPEG (.mpeg)

Otherwise your format is not supported by ROMEO.
