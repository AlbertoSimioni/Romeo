2. BASIC NOTIONS

Basic notions included in this page are indispensable for users who want to start interacting with ROMEO. After understanding them all, users will have the knowledge required for this program.

2D

Identifies images type not time-dependent having two dimensions which correspond to x-axis and to y-axis.

2D-t (with time)

Identifies time-dependent images type having two dimensions which correspond to x-axis and to y-axis. The third dimension is time.

3D

Identifies images type not time-dependent having three dimensions which corre- spond to x-axis, to y-axis and to z-axis.

3D-t (with time)

Identifies time-dependent images type having three dimensions which correspond to x-axis, to y-axis and z-axis. The fourth dimension is time.

Analyze

In the field of biomedical imaging, this term indicates a file format used for storing MRI data. It consists of two files: the actual image file (.img) and its header (.hdr).

Clustering

Technique that groups a set of objects in such a way that objects in the same group (called cluster) are more similar (in some sense or another) to each other than to those in other groups (clusters).

Clustering algorithm

Function that applies a clustering technique to data that can be original images or extracted features.

Data

Identifies biomedical image to be analyzed.

Dataset

Archive of Data. It is organized in subjects, each one with one image to analysed. For each dataset one or more protocols can be associated.

Feature

Information extracted from the image, not present originally.

Feature extractor

Function that calculates a feature of interest from the data.

GUI (Graphical User Interface)

Set of elements and structures that allows the user to interact with the machine by manipulating conventional graphic objects.

Library

In computer science, a library is a collection of implementations of behaviors, written in terms of a language, that has a well-defined interface by which the behavior is invoked. This means that as long as a higher level program uses a library to make system calls, it does not need to be re-written to implement those system calls over and over again. In addition, the behavior is provided for reuse by multiple independent programs. A program invokes the library-provided behavior via a mechanism of the language.

Mask

A binary image, with same format of data, which is used to limit the analysis to the image elements of interest (the data elements whose corresponding mask values are zero are discarded from the analysis).

NIfTI

NIfTI is a new Analyze-style data format, proposed by the NIfTI Data Format Working Group as a short-term measure to facilitate inter-operation of functional MRI data analysis software packages.

Official protocol

Identifies a definitive protocol.

Protocol

Set of one or more feature extractors and/or one clustering algorithm. When a protocol is applied to a dataset, its execution must involve all selected subjects. Executionâ€™s results are stored in a folder of file systems.

Shared library

Library that is loaded by program when it starts. When a shared library is installed properly, all programs that start afterwards automatically use the new shared library.

Subject

A single instance of the dataset, to whom the data belongs to.

Test protocol

Identifies a not definitive protocol used to verify that results produced by its application are those expected. A test protocol can be transformed into an official one.
